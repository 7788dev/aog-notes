# 写作指南

本文介绍如何在 AOG Notes 中编写和组织笔记。

## 目录结构

笔记存放在 `data/` 目录下，使用文件夹进行分类：

```
data/
├── 01-快速开始/
│   ├── 01-介绍.md
│   ├── 02-安装.md
│   └── 03-配置.md
├── 02-使用指南/
│   ├── 01-写作指南.md
│   └── 02-Markdown语法.md
└── 03-进阶/
    └── 01-自定义主题.md
```

## 命名规则

### 文件夹命名

格式：`序号-分类名称`

- 序号用于排序，建议使用两位数字（01、02、03...）
- 分类名称支持中文和英文
- 序号在显示时会自动去除

### 文件命名

格式：`序号-标题.md`

- 同样使用序号控制排序
- 标题将作为笔记的默认标题
- 文件名会自动转换为 URL 友好的 slug

## Front Matter

可以在 Markdown 文件开头使用 YAML 格式的 Front Matter 设置元数据：

```yaml
---
title: 自定义标题
---
```

如果不设置 title，将使用文件名作为标题。

## 子分类

支持在分类下创建子文件夹：

```
data/
└── 01-编程语言/
    ├── 01-JavaScript/
    │   ├── 01-基础语法.md
    │   └── 02-ES6特性.md
    └── 02-Python/
        └── 01-入门教程.md
```

子分类会在侧边栏中以折叠形式展示。

## 最佳实践

1. **保持结构清晰** - 每个分类不超过 10-15 篇笔记
2. **合理使用序号** - 按照阅读顺序或重要程度排序
3. **标题简洁明了** - 标题应该能概括笔记内容
4. **善用子分类** - 内容较多时使用子分类组织
