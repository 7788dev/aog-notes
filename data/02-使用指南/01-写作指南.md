# 写作指南

本文介绍如何在 AOG Notes 中编写和组织笔记。

## 目录结构

笔记存放在 `data/` 目录下，使用文件夹进行分类：

```
data/
├── 01-快速开始/
│   ├── 01-介绍.md
│   ├── 02-安装.md
│   └── 03-配置.md
├── 02-使用指南/
│   ├── 01-写作指南.md
│   └── 02-Markdown语法.md
└── 03-进阶/
    └── 01-自定义主题.md
```

## 命名规则

### 文件夹命名

格式：`序号-分类名称`

- 序号用于排序，建议使用两位数字（01、02、03...）
- 分类名称支持中文和英文
- 序号在显示时会自动去除（可在配置中关闭）

### 文件命名

格式：`序号-标题.md`

- 同样使用序号控制排序
- 标题将作为笔记的默认标题
- 文件名会自动转换为 URL 友好的 slug（中文转拼音）

## Front Matter

可以在 Markdown 文件开头使用 YAML 格式的 Front Matter 设置元数据：

```yaml
---
title: 自定义标题
tags:
  - JavaScript
  - 前端
  - 教程
date: 2024-12-19
---

# 文章内容从这里开始
```

### 支持的字段

| 字段 | 说明 | 示例 |
|------|------|------|
| `title` | 自定义标题（可选） | `title: 我的笔记` |
| `tags` | 标签列表 | `tags: [JavaScript, React]` |
| `date` | 发布日期 | `date: 2024-12-19` |

如果不设置 title，将使用文件名作为标题。

## 标签系统

在 Front Matter 中添加标签：

```yaml
---
tags:
  - JavaScript
  - React
  - 前端开发
---
```

标签功能：
- 显示在文章详情页标题下方
- 点击标签可以搜索相关文章
- 支持多个标签

## 子分类

支持在分类下创建子文件夹：

```
data/
└── 01-编程语言/
    ├── 01-JavaScript/
    │   ├── 01-基础语法.md
    │   └── 02-ES6特性.md
    └── 02-Python/
        └── 01-入门教程.md
```

子分类会在侧边栏中以折叠形式展示，点击分类名称可以展开/收起。

## 阅读时间

启用 `features.readingTime` 后，系统会自动计算每篇文章的预估阅读时间：

- 默认阅读速度：200 字/分钟
- 可在 `advanced.wordsPerMinute` 中调整
- 显示在文章详情页标题下方

## 图片使用

### 本地图片

将图片放在 `public/` 目录下：

```markdown
![图片描述](/images/my-image.png)
```

### 外部图片

直接使用完整 URL：

```markdown
![图片描述](https://example.com/image.png)
```

## 最佳实践

1. **保持结构清晰** - 每个分类不超过 10-15 篇笔记
2. **合理使用序号** - 按照阅读顺序或重要程度排序
3. **标题简洁明了** - 标题应该能概括笔记内容
4. **善用子分类** - 内容较多时使用子分类组织
5. **添加标签** - 便于读者发现相关内容
6. **设置日期** - 帮助读者了解内容时效性

## 热更新

开发模式下，修改 Markdown 文件后页面会自动刷新，无需手动重启服务器。
